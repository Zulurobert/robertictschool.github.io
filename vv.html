data validation

Data validation is the process of checking input data by the computer. Or rather data validation is the process of checking data that has been input by the computer

What is the purpose of data validation?

The purpose of data validation is to prevent data corruption, or security vulnerability, which is caused by data validation.

Types of validation checks.

1: Format check: it checks that data is in the right format. For example, the date.
2: Data type check: it checks the data type given. For example, if numeric data is entered where alphabetical data is needed, an error message will appear.
3: Range check: This checks that the data lies between a specific range. For example, the date of a person’s birth should lie between 1 and 12.
4: Spelling check: This looks for spelling and grammar errors.
5: Consistency checks: This checks that data corresponds. For example, if the title Mr is used, then the gender should be male.
6: Check digits: A check digit is a form of redundancy check used for error detection. Or, a check digit is an additional digit located at the far right side of a bar code, and calculated using a modulus 11 method

Let us now look the process for generating check digits.

1: Get the account number, or key field.
2: using an appropriate modulus, calculate the check digit.
3: add the check digit to the original account number.
4: every time the account number is edited on the computers system, the system re-calculates the check digit.

Let us now look at how to calculate a check digit.

1: write down the account number or key field.
2: Starting from the right to the left, write numbers beginning from 2 to 7, below the account number. These numbers are called the weighted factors. You should note that once you reach 7, u start from 2 again.
3: multiply the figures in the account number, with the figures in the weighted factors.
4: add the results of the multiplications together.
5: divide the result of the addition by 11.
6: subtract the remainder of the division from 11. And that gives you the check digit.

Note that:

1: if the remainder is 0, then the check digit is 0.
2: If the remainder is 1, then the code becomes void.
3: the check digit cannot be two numbers. For example, it cannot be 10.
4: weighted factors are from 2 to 7. The weighted factor cannot be 1, 8, 9, etc.

data verification

Verification means to double click

Let us now look at the verification process.

1: Data is given to an operator to enter.
: The same data is given to a second person to re-enter. This second person is called a verifier.
3: The verifier starts entering the data afresh.


Let us now look at the types of errors detected during verification.

They are four types of errors detected during verification. Namely, transposition errors, double transposition errors, errors of omission, and transcription errors.

Let us now look at them in detail.

1: Transposition error: A transposition error is an error caused by switching the positions of two adjacent digits of an amount while posting a transaction. For example, the figure 132000, will be written as 123000.
2: Double transposition errors. A double transposition error is an error caused by switching the positions of more than two numbers of an amount while posting a transaction.
3: Error of omission. An error of omission, is a transaction that has been omitted from the books of accounts all together.
4: Transcription error. A transcription error is an error, whereby the data is changed all together.

What is data integrity?

Thus refers to maintaining, and assuring, the accuracy, and consistency, of data.
